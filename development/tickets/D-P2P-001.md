## Problem

`csv-p2p/src/proof_delivery.rs` has a comment "// For now, we don't have author info in the proof bundle itself" indicating that proof delivery cannot attribute proofs to authors because the proof bundle lacks author metadata.

`csv-p2p/src/nostr.rs` has stub implementations for proof broadcast and subscription when the nostr feature is disabled:
- `Proof broadcast (stub - nostr feature disabled)`
- `Proof subscription channel created (stub - nostr feature disabled)`

## Why it matters

The P2P layer is responsible for proof distribution. Stubs mean:
- Proof delivery cannot attribute proofs (no author info)
- Nostr-based proof broadcast is a no-op when the feature is disabled
- There's no fallback mechanism for proof distribution

## Task

1. For `proof_delivery.rs`: Either add author info to the proof bundle type, or document that author attribution is out of scope and return a typed error when author info is required.

2. For `nostr.rs`: Replace stub implementations with proper error handling. When the nostr feature is disabled, return `Err(P2pError::NostrNotConfigured)` instead of logging a stub message.

## Acceptance criteria

- [ ] Proof delivery either has author info or returns a clear typed error
- [ ] Nostr stubs return proper errors instead of logging stub messages
- [ ] All "For now" and "stub" comments are removed
- [ ] `cargo check -p csv-p2p` passes
- [ ] `cargo test -p csv-p2p` passes

## Notes

Check if the proof bundle type (`ProofBundle`) can be extended with author metadata without breaking the wire format. If not, author attribution may need to be handled at a higher level.
